01 — Overview

Project Definition

A clean, 3D-enhanced, fully CMS-managed animation artist portfolio. Inspired by the minimal editorial layout of natalieroseyoung.com but elevated with live Three.js 3D scenes, smooth scroll animations, and a private admin dashboard for managing all artwork, categories, and site content.

🎯 Core Purpose
Showcase animation work to studios, directors, and clients. Drive commissions and job inquiries. Position the artist as premium talent in the industry.
👤 Primary Users
Visitors: Studio recruiters, indie directors, fans, collaborators browsing the portfolio.

Admin: The artist — managing projects, uploading work, updating bio, reading contact form messages.
📦 Core Value
A portfolio that feels like a living, breathing art installation — not a static webpage. The 3D scene and animations communicate the artist's craft before a single image loads.
✅ Must Have (V1)
Home with live 3D scene → Portfolio grid with category filtering → Individual project detail pages → Character Design / Props / Comics / Figures / Sketchbook category pages → About page → Contact form → Admin CMS dashboard → Image/video upload → Mobile responsive → SEO metadata
🔮 Nice to Have (V2)
Blog/journal section → Password-protected client proofing gallery → Animation reel embed with custom player → Newsletter subscribe → Multi-language support → Dark/light mode toggle → Project collaboration request form
02 — Tech Stack

Technology Choices

Every choice below is intentional. The goal is the fastest build path to a production-grade result with the lowest operational burden.

Frontend
▲
Next.js 14 (App Router)
Core framework — pages, routing, SSR, API routes
App Router enables server components for faster page loads. Built-in Image optimization. Zero-config deployment to Vercel.
TS
TypeScript
Type safety across the entire codebase
Catches bugs before runtime. Essential for a CMS with complex content types and API responses.
🌊
Tailwind CSS + CSS Custom Properties
Utility styling + design token variables
Design tokens (colors, fonts, spacing) live in CSS vars; Tailwind handles layout and responsive behavior. No runtime JS styling overhead.
🔵
Three.js + React Three Fiber (R3F)
3D canvas background and decorative 3D elements
R3F integrates Three.js natively into React's component tree. Drei helpers accelerate common patterns (orbit, environment, effects).
✨
Framer Motion
Page transitions, scroll reveal, micro-interactions
Best-in-class React animation library. useInView hook handles scroll-triggered animations. Layout animations for portfolio filtering.
🎞
GSAP (ScrollTrigger)
Complex timeline animations and scroll-driven effects
Handles the marquee strip, hero stagger reveals, and any timeline-based sequences that Framer Motion can't handle cleanly.
Backend & Data
🗄
Supabase (PostgreSQL)
Database + auth + real-time + file storage
Managed PostgreSQL with a generous free tier. Built-in auth for the admin CMS. Storage bucket for project images and videos.
🖼
Cloudinary
Image & video CDN with on-the-fly transforms
Automatic WebP conversion, responsive srcsets, video transcoding, and global CDN. Offloads all media serving from the app.
📧
Resend (Email)
Contact form delivery + admin notifications
Modern developer-first email API. React Email for templating. Reliable deliverability.
🔐
NextAuth.js v5
Admin authentication
Credentials provider for username/password admin login. JWT sessions. Protected API routes middleware.
DevOps & Tooling
▲ Vercel
Deployment, edge CDN, preview environments on every PR
🐙 GitHub
Version control, PR workflow, GitHub Actions CI
🧪 Vitest
Unit tests for utilities, API handlers, and validation
🎭 Playwright
E2E tests for critical flows: contact form, admin login, project upload

06 — Motion

All 18 Animations Specified

Every animation defined with library, trigger, duration, and easing. Nothing left to guess.

Three.js / R3F — 3D Scene
HeroScene — Live 3D Background Canvas
Full-viewport Three.js canvas rendered behind the hero section. Contains: icosahedron (wireframe + solid), torus, octahedron, torus knot, floating sphere, and 200-point particle field. All objects respond to mouse position via smooth lerp on camera X/Y. Objects float up/down on sinusoidal time-based offset. Scene loads lazily after hero text is visible.
Library: React Three Fiber + Drei
Trigger: Page load (lazy, 300ms delay)
Mouse: Camera lerp 0.04 factor
Float: sin(t × 0.4–0.7) × 0.3–0.6 amplitude
Performance: devicePixelRatio capped at 2, pauses when tab hidden
Framer Motion — Scroll & Page Animations
Hero Stagger Reveal
Eyebrow, title line 1, title line 2, subtitle, description, and buttons each animate sequentially on page load.
Trigger: Page mount
From: y:30, opacity:0
To: y:0, opacity:1
Stagger: 0.1s per element
Duration: 0.8s ease-out
Scroll Reveal (Universal)
Any element with the RevealWrapper component animates in when it enters the viewport.
Trigger: IntersectionObserver threshold:0.15
From: y:40, opacity:0
To: y:0, opacity:1
Duration: 0.8s var(--ease-smooth)
Once: true (no re-trigger)
Portfolio Filter (Layout Animation)
When a category filter is selected, cards animate out/in using Framer Motion's AnimatePresence + layout prop so positions smoothly transition.
Library: Framer Motion AnimatePresence
Filtered out: scale:0.95, opacity:0
Filtered in: scale:1, opacity:1, stagger:0.04s
Layout: Framer Motion layout prop
Work Card Hover
On card hover: overlay fades in (opacity 0→1), inner image scales up slightly, card lifts (translateY -4px). On leave: all reversed.
Overlay: opacity 0→1, 0.4s ease
Image: scale 1→1.04, 0.6s ease
Card lift: translateY 0→-4px, 0.3s ease
Page Transitions
Between pages: outgoing page fades + slides up, incoming page fades + slides up from below. Handled by PageTransition wrapper in layout.
Exit: y:0→-20, opacity:1→0, 0.3s
Enter: y:20→0, opacity:0→1, 0.5s
Easing: var(--ease-smooth)
Skill Bar Fill
When About section enters viewport, each skill bar fills from left to right with a staggered delay. Numbers count up simultaneously.
Trigger: IntersectionObserver
Transform: scaleX(0→var(--width))
Duration: 1.2s cubic-bezier(.4,0,.2,1)
Stagger: 120ms per bar
GSAP — Marquee & Complex Timelines
Marquee Strip (GSAP)
Infinite horizontal scroll of skills text. Cloned to fill width seamlessly. Pauses on hover.
Library: GSAP gsap.to()
Duration: 30s linear infinite
Clone: Duplicate inner to fill seamlessly
Hover: gsap.to timeScale(0.1)
Stat Count-Up
Number stats animate from 0 to their final value when the about section scrolls into view.
Library: useCountUp custom hook
Trigger: IntersectionObserver
Easing: GSAP Power2.easeOut
Duration: 1.8s
CSS — Cursor, Progress, Micro-interactions
Custom Cursor
Dot follows mouse precisely. Ring lags behind with lerp. Ring scales up (2×) and changes color on hoverable elements.
Dot: Direct mouse coords
Ring: rAF lerp factor 0.14
Hover: ring scale 1→1.6
Scroll Progress Bar
Thin gold bar at top of page that fills as user scrolls. Uses CSS custom property updated by JS scroll listener.
Method: CSS width% from scrollY
Update: requestAnimationFrame
Color: gold → light-gold gradient
Rotating Badge
Circular badge in hero rotates 360° on a 20s loop using CSS animation.
Method: CSS @keyframes rotate
Duration: 20s linear infinite
Inner: Counter-rotates to stay readable

Admin Authentication Flow

Auth Flow
// 1. Admin visits /admin → middleware.ts checks session
// 2. No session → redirect to /login
// 3. Login form POSTs credentials to /api/auth/callback/credentials
// 4. NextAuth validates against env vars (single admin user)
// 5. Valid → JWT session created, redirect to /admin
// 6. All /api/* admin routes check getServerSession()

// middleware.ts

11 — Risks

Risks & Key Decisions

RiskLikelihoodImpactMitigation
Three.js causes poor Lighthouse performance scoreMediumHighDynamic import with ssr:false, defer canvas load until after LCP, offer prefers-reduced-motion fallback
Three.js crashes on low-end Android devicesMediumMediumDetect WebGL support on client, show gradient fallback if unavailable
Cloudinary upload size limits block video uploadsLowMediumUse Cloudinary's chunked upload API for files >10MB. Set 50MB cap in API.
Contact form spam despite rate limitingMediumLowAdd honeypot field, reCAPTCHA v3 as V2 enhancement, and email keyword filtering
Admin password leak → portfolio vandalizedLowHighbcrypt hash password in DB (not env var), add 2FA support in V2, IP whitelist /admin if needed
Supabase free tier connection limit hitLowMediumUse connection pooling via Supabase's pooler endpoint for serverless functions
ISR stale cache shows old content after CMS updateMediumMediumCall revalidatePath() on every CMS write. Add on-demand revalidation route.
Key Decisions Requiring Confirmation
Decision 1: CMS Complexity
Option A — Custom CMS (this plan): Built into the app, full control, no monthly cost.

Option B — Headless CMS (Sanity/Contentful): Better editing UX, richer content modeling, $0–$50/month.

Recommendation: Option A for a solo artist portfolio. Add Sanity in V2 if editing friction is too high.
Decision 2: 3D Library
Option A — React Three Fiber (this plan): React-native, cleaner component model, full ecosystem.

Option B — Vanilla Three.js: Smaller bundle if R3F overhead matters, direct control.

Recommendation: R3F — the Drei helpers (OrbitControls, Sparkles, etc.) save days of work.
